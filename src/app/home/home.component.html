1. Eager Loading

Eager loading is loading modules before application starts.

To load a feature module eagerly, we need to import that module in application module i.e. AppModule using imports metadata of @NgModule decorator.

Eager loading is good for small applications because at the first hit of the application all the modules are loaded and all the required dependencies are resolved. Now the subsequent access to the application will be faster .


2. Lazy Loading

Lazy loading is loading modules on demand.

To load a feature module lazily, we need to load it using loadChildren property in route configuration and that feature module must not be imported in application module.

Lazy loading is useful when the application size is growing. In lazy loading, feature module will be loaded on demand and hence application start will be faster.

If you generate new module with routing you must create lazy loading in app component after that you can use router-outlet in generated module
